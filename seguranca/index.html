Security

  • Home
  • Security

All the information contained in the website, as well as the conversations that are carried out via chat, are ensured via SSL (Digital Certificate), ensuring, what is most important, the encryption in the data of the customers. In order for you to understand a little more how each one works, here is a brief explanation of the two forms of security.

SSL - Secure Socket Layer (SSL) is a global standard in security technology developed by Netscape in 1994. It creates an encrypted channel between a web server and a browser (browser) to ensure that all transmitted data is confidential and secure. It is the famous "golden lock" "  that appears in browsers when we are accessing a secure website.

When choosing to activate SSL on your web server you will have to answer some questions about the identity of your site (ex. URL) and your company (ex. Company Name and address). Your web server will then create two cryptographic keys - the Private Key and the Public Key. Your Private Key is not named like that for nothing, it must be kept private and secure. The Public Key, on the other hand, does not need to be secret and must be placed in the CSR (Certificate Signing Request), a data file containing the details of the website and the company. You must send this CSR through the request form on our website, your data will be validated and if they are correct your digital certificate will be issued. Our web server will associate the issued certificate with your Private Key.

Encryption: when data is encrypted, it means that an algorithm has been applied to encode it so that it loses its original form and therefore cannot be read. Data can be decrypted back to its original form by applying a specific decryption key. Encryption techniques are an important part of data security as they protect sensitive information from threats including malware exploitation and unauthorized access by third parties. Data encryption is a versatile security solution: it can be applied to one piece of data (such as a password), or more broadly to all data in a file, or even to all data contained on storage media.
